Here's a concise, clear prompt for Replit AI:

---

## **FEATURE REQUEST: Azure DevOps Backlog Context Awareness**

### **OVERVIEW**

Enhance the artifact generation workflow to fetch and analyze existing Azure DevOps backlog items before generating new artifacts. This ensures the AI avoids duplicates, references existing work items, and intelligently aligns new artifacts with the current backlog structure.

---

### **REQUIREMENTS**

### **1. Fetch Existing ADO Backlog Context**

**When:** Before starting the conversation on Workflow page (after compliance guidelines selection, if any)

**What to Fetch:**
Using the ADO credentials from Golden Repo, fetch existing backlog items:

**API Calls:**

```
GET {orgUrl}/{project}/_apis/wit/wiql?api-version=7.0

Query Body:
{
  "query": "SELECT [System.Id], [System.Title], [System.Description], [System.WorkItemType], [System.State], [Microsoft.VSTS.Common.Priority], [System.Tags], [System.Parent] FROM WorkItems WHERE [System.TeamProject] = '{project}' AND [System.WorkItemType] IN ('Epic', 'Feature', 'User Story', 'Task') ORDER BY [System.CreatedDate] DESC"
}
```

Then fetch details for each work item:
```
GET {orgUrl}/{project}/_apis/wit/workitems?ids={comma-separated-ids}&api-version=7.0
```

**Data to Extract:**
- Work Item ID
- Title
- Work Item Type (Epic, Feature, User Story, Task)
- Description (first 200 characters)
- State (New, Active, Resolved, Closed)
- Priority (1-4)
- Tags
- Parent Work Item ID (to understand hierarchy)

### **3. Inject Backlog Context into AI System Prompt**

Add a new section to the system prompt BEFORE the conversation starts:

```
## EXISTING AZURE DEVOPS BACKLOG CONTEXT

⚠️ IMPORTANT: The project already has existing backlog items in Azure DevOps. You MUST be aware of this context to avoid duplicates and align new work properly.

### Backlog Summary:
- **Total Epics:** ${summary.totalEpics} (${summary.activeEpics} Active)
- **Total Features:** ${summary.totalFeatures} (${summary.activeFeatures} Active)
- **Total User Stories:** ${summary.totalUserStories} (${summary.activeUserStories} Active)

### Existing Epics:
${epics.map(epic => `
**Epic #${epic.id}: ${epic.title}**
- State: ${epic.state} | Priority: ${epic.priority}
- Description: ${epic.description}
- Tags: ${epic.tags.join(', ')}
- Has ${epic.features.length} child feature(s)
`).join('\n')}

### Existing Features:
${features.map(feature => `
**Feature #${feature.id}: ${feature.title}** (under Epic #${feature.parentEpic})
- State: ${feature.state} | Priority: ${feature.priority}
- Description: ${feature.description}
- Has ${feature.userStories.length} user story/stories
`).join('\n')}

### Existing User Stories (Recent):
${userStories.slice(0, 20).map(story => `
**Story #${story.id}:** ${story.title} (Feature #${story.parentFeature})
- State: ${story.state} | Priority: ${story.priority}
`).join('\n')}

## HOW TO USE THIS CONTEXT:

### 1. AVOID DUPLICATES
- Before suggesting a new Epic/Feature/Story, CHECK if something similar already exists
- If similar work exists, mention it: "I see you already have an Epic called 'Customer Portal Enhancement' (#123). Should we add to that or create something new?"

### 2. INTELLIGENT ALIGNMENT
- When creating new User Stories, suggest placing them under existing relevant Features
- Example: "This new story about user registration fits well under your existing Feature 'User Onboarding Flow' (#456). Shall I add it there?"

### 3. REFERENCE EXISTING ITEMS
- Use work item IDs when referencing: "This connects to Story #789"
- Acknowledge existing work: "Building on your existing 'Customer Portal' epic..."

### 4. IDENTIFY GAPS
- Notice patterns and suggest missing pieces
- Example: "I notice your 'User Onboarding Flow' feature has stories for registration but nothing about email verification. Should we add that?"

### 5. RESPECT HIERARCHY
- Understand Epic → Feature → User Story → Task hierarchy
- Don't create orphaned items; always suggest proper parent relationships

### 6. CONSIDER STATE & PRIORITY
- Be aware of what's already Active vs. New vs. Closed
- Align priorities with existing items
- Example: "Your existing onboarding stories are Priority 2. Should these new ones match that priority?"

### 7. CONVERSATIONAL AWARENESS
During the conversation, proactively:
- Ask if new requirements relate to existing epics/features
- Suggest consolidation when appropriate
- Mention existing work that might be impacted
- Recommend updates to existing items if relevant

Example Dialogue:
User: "We need to build a user registration feature"
Agent: "I noticed you already have a Feature called 'User Onboarding Flow' (#456) which includes registration. Would you like me to:
1. Add new user stories under that existing feature?
2. Create a completely separate feature for a different type of registration?
3. Review and refine the existing registration stories?"

---

## BACKLOG CONTEXT VALIDATION RULES:

Before generating final artifacts:
- [ ] Checked for duplicate or overlapping epics/features
- [ ] Identified appropriate parent items for new stories
- [ ] Referenced existing work item IDs where relevant
- [ ] Suggested consolidation opportunities
- [ ] Aligned priorities and tags with existing patterns
- [ ] Flagged potential impacts on existing backlog items

If user requirements clearly overlap with existing work, ASK before creating duplicates.
```

Should we address any of these gaps in what we're building today?"

### **7. Error Handling**

```javascript
// Handle empty backlog
if (adoBacklogContext.summary.totalEpics === 0) {
  message.info('No existing backlog items found. Starting fresh!');
}

// Handle fetch failures
try {
  const backlogContext = await fetchADOBacklog(adoCredentials);
} catch (error) {
  console.error('Failed to fetch backlog:', error);
  message.warning('Could not load existing backlog. Proceeding without context.');
  // Continue without backlog context - don't block workflow
}

// Handle large backlogs (>200 items)
if (totalItems > 200) {
  message.info('Large backlog detected. Showing most recent 200 items.');
  // Limit to recent/active items
}
```

2. **Existing Work:** Project with 5 epics, 15 features → AI references them during conversation

4. **Gap Identification:** Backlog missing obvious stories → AI suggests filling gaps
