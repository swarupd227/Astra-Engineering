TROUBLESHOOTING: Wiki Docs Not Pushed to Azure DevOps
MOST COMMON ISSUES
1. Wiki API Endpoint Differences
Azure DevOps has two types of wikis with different APIs:
Project Wiki (what you're using):

URL pattern: .../_wiki/wikis/{ProjectName}.wiki/...
API: POST {orgUrl}/{project}/_apis/wiki/wikis/{wikiIdentifier}/pages?path={pagePath}&api-version=7.0

Code Wiki (wiki in a Git repo):

Different API: Uses Git repository APIs

Check which type you have:
javascript// Verify wiki type
GET https://dev.azure.com/DevXPlatform/NousAugmentedDevX/_apis/wiki/wikis?api-version=7.0

// Response will show wiki type and identifier

2. Missing Wiki Identifier
The code might be using the wrong wiki identifier.
Your wiki URL shows: NousAugmentedDevX.wiki
Check if code is using:
javascript// ❌ Wrong - using project name instead of wiki identifier
const wikiId = "NousAugmentedDevX"; 

// ✅ Correct - should be
const wikiId = "NousAugmentedDevX.wiki";

3. Incorrect API Endpoint
Correct endpoint for creating/updating wiki pages:
javascript// Create new page
PUT https://dev.azure.com/DevXPlatform/NousAugmentedDevX/_apis/wiki/wikis/NousAugmentedDevX.wiki/pages?path={pagePath}&api-version=7.0

// Update existing page  
PUT https://dev.azure.com/DevXPlatform/NousAugmentedDevX/_apis/wiki/wikis/NousAugmentedDevX.wiki/pages?path={pagePath}&api-version=7.0

// Headers
{
  "Authorization": "Basic " + btoa(":" + patToken),
  "Content-Type": "application/json"
}

// Body
{
  "content": "# Wiki Page Content\n\nMarkdown content here..."
}
Note: For updates, you need If-Match header with eTag:
javascript// Get current page first to get eTag
GET .../pages?path=/PageName

// Then update with eTag
PUT .../pages?path=/PageName
Headers: {
  "If-Match": eTagFromGetRequest
}

4. PAT Token Permissions
Required permissions for Wiki operations:
Check if your PAT token has:

✅ Code: Read & Write (if using Code Wiki)
✅ Wiki: Read & Write (for Project Wiki)

Verify token permissions:

Go to Azure DevOps → User Settings → Personal Access Tokens
Find your token → Edit
Ensure "Wiki (Read, write, & manage)" is checked


5. Page Path Format Issues
Common mistakes:
javascript// ❌ Wrong - missing leading slash
path: "Overview-and-Vision"

// ❌ Wrong - using backslashes
path: "\Overview-and-Vision"

// ❌ Wrong - spaces not encoded
path: "/Overview and Vision"

// ✅ Correct formats
path: "/Overview-and-Vision"           // Top-level page
path: "/Parent-Page/Child-Page"        // Nested page
path: "/Overview%20and%20Vision"       // Spaces encoded
