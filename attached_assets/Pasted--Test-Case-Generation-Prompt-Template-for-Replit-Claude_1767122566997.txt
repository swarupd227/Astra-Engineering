## TEST CASE GENERATION REQUIREMENTS:

Generate detailed test cases for each of the following categories:

### 1. FUNCTIONAL TEST CASES (Positive Scenarios)
- Cover all acceptance criteria with happy path scenarios
- Validate core business logic and workflows
- Test all input combinations that should work
- Verify expected outputs and system behavior
- Include data validation for valid inputs

### 2. NEGATIVE TEST CASES
- Invalid input handling (empty, null, special characters, SQL injection attempts)
- Boundary violations (exceed max/min limits)
- Invalid data formats (wrong date format, invalid email, etc.)
- Unauthorized access attempts
- Missing mandatory fields
- Invalid state transitions
- Timeout and error handling scenarios
- Concurrent operation conflicts

### 3. EDGE CASE TEST CASES
- Boundary value analysis (min, max, min-1, max+1)
- Empty datasets and single record scenarios
- Maximum capacity testing
- Leap year, daylight saving, timezone edge cases
- Unicode and special character handling
- Very long strings and large numbers
- First and last record processing
- Simultaneous operations at boundaries

### 4. WORKFLOW/INTEGRATION TEST CASES
- End-to-end workflow validation
- Multi-step process completion
- System integration points
- Data flow between modules
- State transitions across workflow stages
- Rollback and recovery scenarios
- Batch processing workflows
- Notification and alert workflows

### 5. SECURITY TEST CASES
- Authentication validation (valid/invalid credentials)
- Authorization and role-based access control
- Session management (timeout, concurrent sessions)
- Data encryption verification
- Input sanitization (XSS, SQL injection prevention)
- Sensitive data masking
- Audit trail integrity
- API security (token validation, rate limiting)
- CSRF protection
- Password policy enforcement

### 6. ACCESSIBILITY TEST CASES (WCAG 2.1 Compliance)
- Screen reader compatibility
- Keyboard navigation (Tab, Enter, Escape)
- Focus indicators and tab order
- Color contrast validation (4.5:1 ratio)
- Alt text for images and icons
- Form label associations
- Error message accessibility
- Skip navigation links
- Responsive design for assistive devices
- ARIA attributes validation

## 🚀 REPLIT INTEGRATION TIPS

### Setting Up Claude in Replit:

1. **Create a new Replit project** (Python or Node.js recommended)

2. **Install Anthropic SDK:**
   ```bash
   # For Python
   pip install anthropic
   
   # For Node.js
   npm install @anthropic-ai/sdk
   ```

3. **Set API Key in Replit Secrets:**
   - Go to Tools > Secrets
   - Add key: `ANTHROPIC_API_KEY`
   - Add your Claude API key as value

4. **Sample Python Code for Replit:**

```python
import anthropic
import os

client = anthropic.Anthropic(
    api_key=os.environ.get("ANTHROPIC_API_KEY")
)

def generate_test_cases(user_story, acceptance_criteria, test_type="all"):
    
    prompt = f"""
    You are an expert QA Engineer. Generate comprehensive {test_type} test cases.
    
    USER STORY:
    {user_story}
    
    ACCEPTANCE CRITERIA:
    {acceptance_criteria}
    
    Generate detailed test cases with:
    - Test Case ID
    - Description
    - Preconditions
    - Test Steps
    - Expected Results
    - Priority
    """
    
    message = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=8000,
        messages=[
            {"role": "user", "content": prompt}
        ]
    )
    
    return message.content[0].text

# Example Usage
user_story = """
As a Policy Administrator,
I want to automatically generate renewal notices for policies approaching expiration,
So that policyholders receive timely notifications and can renew their coverage without lapse.
"""

acceptance_criteria = """
1. System identifies all policies expiring within the next 60 days
2. Renewal premium is calculated based on current rating factors
3. Renewal notice is generated with updated premium and coverage details
4. System sends notice via preferred communication channel
5. Policy status updates to 'Pending Renewal'
"""

# Generate all test case types
test_types = ["functional", "negative", "edge case", "workflow", "security", "accessibility"]

for test_type in test_types:
    print(f"\n{'='*50}")
    print(f"GENERATING {test_type.upper()} TEST CASES")
    print('='*50)
    result = generate_test_cases(user_story, acceptance_criteria, test_type)
    print(result)
```

5. **Sample Node.js Code for Replit:**

```javascript
const Anthropic = require('@anthropic-ai/sdk');

const client = new Anthropic({
    apiKey: process.env.ANTHROPIC_API_KEY
});

async function generateTestCases(userStory, acceptanceCriteria, testType = "all") {
    
    const prompt = `
    You are an expert QA Engineer. Generate comprehensive ${testType} test cases.
    
    USER STORY:
    ${userStory}
    
    ACCEPTANCE CRITERIA:
    ${acceptanceCriteria}
    
    Generate detailed test cases with:
    - Test Case ID
    - Description  
    - Preconditions
    - Test Steps
    - Expected Results
    - Priority
    `;
    
    const message = await client.messages.create({
        model: "claude-sonnet-4-20250514",
        max_tokens: 8000,
        messages: [
            { role: "user", content: prompt }
        ]
    });
    
    return message.content[0].text;
}

// Example usage
const userStory = `
As a Policy Administrator,
I want to automatically generate renewal notices for policies approaching expiration,
So that policyholders receive timely notifications.
`;

const acceptanceCriteria = `
1. System identifies policies expiring within 60 days
2. Renewal premium calculated based on rating factors
3. Notice generated with updated premium
`;

generateTestCases(userStory, acceptanceCriteria, "functional")
    .then(result => console.log(result))
    .catch(error => console.error(error));
```

9. Escalation alert triggers if no response is received within 30 days of notice dispatch
10. Renewed policy generates new policy documents with updated effective dates upon payment confirmation

GENERATE TEST CASES FOR:
- Functional (15 minimum)
- Negative (15 minimum)
- Edge Cases (12 minimum)
- Workflow/Integration (10 minimum)
- Security (15 minimum)
- Accessibility (12 minimum)

Use standard test case format with ID, Description, Preconditions, Steps, Expected Results, and Priority.
