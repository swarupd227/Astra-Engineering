SECTION 6: COMPARISON ALGORITHMS (CRITICAL)
=============================================================================

ALGORITHM 1: CELL-BY-CELL COMPARISON
```
Function: compareCells(sourceCell, targetCell, config)

Input: 
  - sourceCell: { value, type, format, formula }
  - targetCell: { value, type, format, formula }
  - config: { numericTolerance, percentageTolerance, caseSensitive }

Process:
  1. Determine cell type (numeric, text, date, boolean, formula)
  2. Apply type-specific comparison:
  
     NUMERIC COMPARISON:
     - If both are numbers:
       - absoluteDiff = |source - target|
       - percentDiff = |source - target| / |source| * 100
       - Pass if absoluteDiff <= numericTolerance OR percentDiff <= percentageTolerance
       - Return: { match: boolean, diff: absoluteDiff, percentDiff: percentDiff }
     
     TEXT COMPARISON:
     - Apply whitespace handling based on config
     - Apply case sensitivity based on config
     - Use Levenshtein distance for similarity score
     - Return: { match: boolean, similarity: percentage, differences: [positions] }
     
     DATE COMPARISON:
     - Parse both dates to timestamp
     - If flexible mode: Compare timestamps only
     - If strict mode: Compare format string as well
     - Return: { match: boolean, daysDiff: number, formatMatch: boolean }
     
     FORMULA COMPARISON:
     - Compare formula strings
     - Compare calculated results separately
     - Return: { formulaMatch: boolean, resultMatch: boolean }

Output:
  {
    match: boolean,
    matchType: 'exact' | 'within_tolerance' | 'mismatch',
    details: { ... type-specific details }
  }
```

ALGORITHM 2: ROW MATCHING
```
Function: matchRows(sourceRows, targetRows, keyColumns)

Input:
  - sourceRows: Array of row objects
  - targetRows: Array of row objects
  - keyColumns: Array of column names to use as keys

Process:
  1. If keyColumns specified:
     - Create composite key from keyColumns for each row
     - Build hash maps for O(1) lookup
     - Match rows by key
  
  2. If no keyColumns (positional matching):
     - Match by row index
     - Flag if row counts differ
  
  3. For unmatched rows:
     - Use fuzzy matching (Jaccard similarity on row values)
     - Suggest potential matches with confidence score

Output:
  {
    matchedPairs: [{ sourceIdx, targetIdx, matchConfidence }],
    unmatchedSource: [rowIndices],
    unmatchedTarget: [rowIndices],
    suggestedMatches: [{ sourceIdx, targetIdx, confidence }]
  }
```

ALGORITHM 3: COLUMN ALIGNMENT
```
Function: alignColumns(sourceColumns, targetColumns)

Input:
  - sourceColumns: [{ name, dataType, sampleValues }]
  - targetColumns: [{ name, dataType, sampleValues }]

Process:
  1. Exact name match (case-insensitive)
  2. Fuzzy name match (Levenshtein distance < threshold)
  3. Data type matching for unmatched columns
  4. Sample value correlation analysis
  5. AI-assisted column mapping for ambiguous cases

Output:
  {
    mappings: [{ sourceCol, targetCol, confidence, method }],
    unmappedSource: [colNames],
    unmappedTarget: [colNames]
  }
```

ALGORITHM 4: STATISTICAL VALIDATION
```
Function: statisticalValidation(sourceData, targetData)

Input:
  - sourceData: Numeric column data
  - targetData: Numeric column data

Process:
  1. Calculate summary statistics for both:
     - Sum, Count, Average, Min, Max, StdDev
  2. Compare statistics within tolerance
  3. Identify outlier differences
  4. Calculate correlation coefficient

Output:
  {
    summaryMatch: boolean,
    statistics: {
      source: { sum, avg, min, max, stdDev },
      target: { sum, avg, min, max, stdDev },
      differences: { sumDiff, avgDiff, ... }
    },
    correlation: number,
    outliers: [{ rowIdx, sourceval, targetVal, zScore }]
  }
```

ALGORITHM 5: PDF COMPARISON
```
Function: comparePDFs(sourcePDF, targetPDF)

Input:
  - sourcePDF: PDF buffer/path
  - targetPDF: PDF buffer/path

Process:
  1. Extract text content using pdf-parse
  2. Extract tables using tabula-js or pdf-table-extractor
  3. For each page:
     - Compare text blocks (position-aware)
     - Compare extracted tables using cell comparison
     - Render page to image for visual comparison
     - Calculate SSIM for visual similarity
  4. Aggregate results across pages

Output:
  {
    pageCount: { source, target },
    textComparison: { matchPercentage, differences },
    tableComparison: { tablesFound, matchResults },
    visualSimilarity: { perPage: [scores], overall: score }
  }
```
