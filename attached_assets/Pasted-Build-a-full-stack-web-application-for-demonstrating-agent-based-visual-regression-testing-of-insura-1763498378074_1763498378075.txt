Build a full-stack web application for demonstrating agent-based visual regression testing of insurance products. This is a DEMO/PROOF-OF-CONCEPT platform meant to be shown to stakeholders. Focus on spectacular visual feedback and real-time agent activity visualization. No error fallbacks needed—assume everything works perfectly.

### Part 2: Real-Time Agent Activity Feed (CENTER STAGE)
This is the HERO of the demo. Build a vertical timeline showing exactly what agents are doing RIGHT NOW.

**Agent Activity Timeline Design:**

- Nodes are color-coded: Gray (pending) → Cyan (in-progress) → Green (completed)

**Each Task Card Shows:**

### Part 6: Backend Logic (Node.js/Express + In-Memory Simulation)

**DO NOT connect to real Figma API or real browsers.** Instead:
- Simulate all API calls with hardcoded success responses
- Simulate task progression with timed delays (each task has a random delay 2-8 seconds)
- Use Server-Sent Events (SSE) or WebSockets to push updates to frontend in real-time

**Express Routes:**
- POST /api/start-demo: Receives Figma URL, website URL, test scope
  - Respond with demo ID and start SSE stream
  - Begin streaming task updates every 500-2000ms
- GET /api/demo/:id/stream: SSE endpoint that streams real-time task updates

**Task Update Format (JSON):**
```json
{
  "taskId": "task-5",
  "taskName": "Capturing Baseline Screenshots",
  "agentName": "Visual Agent",
  "status": "in-progress",
  "progress": 73,
  "details": "Capturing 47 screenshots across Chrome, Firefox, Safari...",
  "timestamp": "2025-11-19T14:23:45Z"
}
```

### Part 9: Application Flow
1. User lands on dashboard
2. Optionally fills in Figma URL and website URL (or uses defaults)
3. Clicks "Begin Testing"
4. Frontend makes POST to /api/start-demo
5. Backend starts SSE stream with task updates
6. Frontend receives updates every 500-2000ms and renders timeline in real-time
7. Tasks progress from "pending" → "in-progress" → "completed"
8. Metrics counter updates as agents "work"
9. Visual comparison preview appears mid-way
10. Final results panel appears when all tasks complete
11. User can optionally click "Run Another Test" to reset

---

### Part 10: Technical Stack
- Frontend: React 18+, TypeScript, Tailwind CSS (or styled-components), Framer Motion
- Backend: Express.js (Node.js), Server-Sent Events (SSE) for real-time updates
- No external API calls to Figma, Playwright, or any real service
- Use mock data and hardcoded responses
- Entire app runs within single Replit workspace
- Deploy to Replit's hosting automatically
